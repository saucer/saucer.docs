# Events

In this chapter, we'll describe the available events and how to use them.

## Basics

Events are simply callbacks which are invoked for specific actions and can, in certain circumstances, even alter the behavior of the application.

### Register Event Callback

To register an event callback simply call the `on` method of your webview.  
You'll need to pass the event-type as the first template parameter and the callback as the function argument.

Every call to `on` will return an `event-id` which can later be used to `unregister` the same callback.

```cpp title="Example Callback (Resize Window)"
smartview.on<saucer::window_event::resize>([](int width, int height)
{
   // ...
});
```

### Unregister Event Callback

As previously mentioned you can unregister an event callback by saving it's `event-id`.  
To unregister, simply call `remove`, with the event-type as the first and the `event-id` as the second argument.

```cpp title="Example Callback Removal"
auto id = smartview.on<saucer::window_event::resize>([](int width, int height)
{
   // ...
});

smartview.remove(saucer::window_event::resize, id);
```

:::tip
If you simply want to remove all event callbacks belonging to a specific event-type you can use the `clear` method.

```cpp title="Clearing all Resize Callbacks"
smartview.clear(saucer::window_event::resize);
```

:::
